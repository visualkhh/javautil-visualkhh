Java Media API

JMF에 대해 소개하기 전에 먼저 Java에서 멀티미디어 데이터를 다루기 위해 발표된 모든 기술들에 간략히 살펴보자. Java에서는 멀티미디어 제어를 위해 Java Media API라는 이름으로 패키지들을 발표했다. Java Media API는 크게 8개의 세부적인 API들로 분류할 수 있다. 세부적인 API의 목록은 표 1과 같다.
표 1과 같이 많은 기술들이 발표되었다. 본 강좌에서는 위와 같은 Java의 Media 처리 기술 중에서도 JMF (Java Media Framework)를 집중적으로 다루어 보겠다. 그럼 JMF는 과연 어떻게 개발되었고, 현재 어떻게 구성되어있는지 살펴보기로 하자.

Java 에서는 멀티미디어를 제어하기 위해 Java Media API를 발표했습니다. Java Media API는 크게 8개의 세부로 분류할 수 있습니다. 세부적인 목록은 다음과 같습니다.

 

1) Java 2D
- 2D 그래픽과 Image의 조작
- Graphics 2D 에서 확장된 Graphics 이용부분

2) Java Media Framework
- 1.0 API 에서는 단순히 동기화된 미디어 데이터에 대한 playback 기능만 제공
- 2.0 API 에서는 오디오 및 비디오데이터에 대한 capture , streaming 기능 제공

3)  Java 3D
- 객체(Object) 기반의 3D 그래픽 runtime 제공
- Java 기술로 제작된 VRML97 reference brower

4)  Java Sound
- software 로 구현된 sound processor 와 MIDI synthesizer 제공
- sound engine(java 2 SDK 1.2)과 sound API (beginning in java 2 SDK 1.3 )제공

5)  Java Speech
- 음성의 인식과 합성 기능 제공

6) Java Advanced Imaging
- 확장된 2D Image processing 기능
- java 2D API Interfaces 들의 구현

7)  Java Telephony
- computer 와 전화 단말기의 통함
- javaphone API 의 하위 API로서 향후 발표될 J2ME 기반의 Mobile information device profile 제공

8)  Java Shared Data Toolkit
- 애플리케이션 기반 Java 기술로서 사용자간 협동 작업을 가능하게 하는 개발 Tool 제공
- Session , URL , Registry 를 통한 객체(Object) 데이터의 공유기능

 

위 사항에서와 같이 JMF 는 Java Media API 의 일부분으로 보아 두시면 좋을것입니다.

 


왜 JMF를 이용해야 하는가?
이제껏 발표된 JDK는 주로 디지털 오디오의 재생 기능을 주된 목적으로 생각하였지만, 디지털 비디오나 심지어 MIDI(Musical Instru ment Digital Interface)에 대한 중요성을 인식하지 못하였다. 또한 이전에 발표된 오디오 관련 API 역시 단순한 디지털 오디오의 재생기능(playback)만을 수행하고, 어떠한 포맷으로도 저장하는 기능을 제공하지 못하였다. 이러한 상황에서 Sun의 Java 진영에서는 당시의 멀티미디어 관련 API들이 불합리하다는 생각을 하게 되었고, 결국 객체지향, 풍부한 기능, 구조적인 멀티미디어 포맷을 지원하며, 단순한 오디오 데이터의 재생 기능 뿐만이 아니라 현재 이용되는 거의 모든 종류의 멀티미디어 데이터를 처리하기 위한 기능을 수행하는 API를 만들게 되었다. 그러한 구조가 바로 Sun의 Java Media API이며 , 대부분의 주된 관심이 Java Media API의 하부 구조인 JMF의 탄생으로 이어지게 된 것이다.


JMF의 세팅 방법
① 먼저 JMF를 설치할 대상 컴퓨터의 하드웨어 및 소프트웨어 사양을 확인한다.
② 만약 이전버젼의 JMF가 설치되어있다면 모두 제거한다.
③ 윈도우 버전의 JMF설치파일을 구동시키면 InstallShield가 구동 되며 설치를 시작한다.
④ CLASSPATH와 PATH를 반드시 확인해 주시기 바란다. 실제로 프로그램을 설치하는 과정에서 InstallShield 가 자동적으로 CLASSPATH와 PATH를 설정해 준다.  만약 설치 후에 JMF의 올바른 설치 확인을 위해 다음과 같은 내용을 확인해야 한다.

set CLASSPATH=%WINDIR%\java\classes\jmf.jar;%WINDIR%
\java\classes\sound.jar;.;%CLASSPATH%
또한 PATH에서도 아래와 같은 사항을 확인해야 한다.

set PATH=%WINDIR%\System32;%PATH% (on Windows NT)
set PATH=%WINDIR%\System;%PATH%   (on Windows 95/98)
올바른 JMF 세팅 확인방법
위와 같은 과정을 통하여 jmf를 모두 설치한 이후에는 SUN의 JMF 사이트를 방문하여 JMF Diagnostics 애플릿 프로그램을 구동시켜 보아야 한다. 실제로 JMF 설치가 모두 올바르게 되었는지를 판별해주는 프로그램이다. 테스트 해볼 프로그램이 있는 http://java.sun.com/products/java-media/jmf/2.1/jmfdiagnostics.html 주소로 방문하여 인터넷 브라우저 역시 JMF를 알맞게 인식하고 있는지 확인해야 한다.
위의 사이트를 방문하면, JMF Diagnostics Applet이 구동 되며, 현재 시스템에 설치된 JMF와 버전, 그리고 Native Libraries를 검사하게 된다. 검사를 마치면 아래와 비슷한 검사 결과를 표시하게 된다.

JMF Diagnostics:
Java 1.1 compliant browser.....Maybe
JMF classes.....Found
JMF Version... 2.1
Win32 Build
Native Libraries Found

만약 위와는 다르게 에러 메시지 혹은 관련 라이브러리 파일을 찾을 수 없다고 메시지가 나오는 경우 상황에 맞게 다시 JMF를 설치하거나 적절한 조치를 취해 주어야 한다.

JMF Diagnostics Applet의 문제 해결 방법

Browser not compatible
JMF는 자바 1.1 호환 브라우저를 요구한다. 추천되는 웹 브라우저는 Netscape Navigator 4.06 or later, MS Internet Explorer 4.0 or later, Sun's Hotjava 1.1.x. 이다. 만약 이전 버전의 브라우저를 사용한다면 최신 버전의 브라우저로 다시 설치해야 한다. 또 하나 주의해야 할 점은 실제 이전 버전의 브라우저를 사용한다면, 최신 버전의 브라우저를 설치한다. 그런 다음에 다시 JMF를 재 설치 해야 한다. 이와 같은 과정을 통해서만 JMF를 설치하는 과정에서 Netscape를 인식해서 필요한 파일들도 다시 정리해 준다.

Classes not found
JMF classes 및 CLASSPATH 환경 변수에 제대로 설정이 되어있지 않을 경우 발생합니다.  CLASSPAT H에서  jmf.jar 를 기록해 주었는지 확인해야 한다. 일반적으로 JMF를 설치하면 이 부분은 자동 세팅 되고 컴퓨터를 재부팅하는데, 혹시라도 이 부분에서 에러메시지가 발생했다면 , 여러분은 수동으로 CLASSPATH를 적어 주어야 한다. 

Libraries not found
JMF를 설치 후에 생성되는 JMStudio 프로그램이 캡처가 안된다고 할 때, 가장 흔하게 나타나는 문제점이 바로 이 부분이다. 이 부분에서 에러가 발생하면 캡처 프로그램을 수행하는 경우 예측하지 못하는 에러가 발생한다. 만약 윈도우나 솔라리스같은 환경에서 캡처 관련 프로그램을 작성한다면 JMF 자체도 역시 Native Version의 JMF를 다운 받아 설치해야 한다.
다음으로 이 에러가 발생하는 이유는 JMF의 lib 디렉토리를 브라우저가 제대로 확인하지 못해서 NativeCode를 찾지 못하기 때문이다. 이러한 라이브러리들은 \lib 밑에 있다. 이 디렉토리를 여러분의 bat 파일에 기록해 주어야 한다. 하지만 문제점이 하나있다. 만약 ‘IE를 사용하고 있고, 분명히 제대로 설정도 다 해주었는데 왜 안되는가' 하는 질문에 대한 답변은 가장 최신 버전의 IE용 java VM을 다시 다운로드 받아야 한다는 것이다(http://www.microsoft.com/java 이곳에서 다시 java VM을 다운로드 받아 설치하면 된다).
